\# NeverLate - MVP Requirements



\## Client

Bladimir Navarro



\## Project

NeverLate



\## MVP Scope

The MVP will focus on the core functionality required to validate the product idea: reminders, documents, checklists, notifications, calendar view, and basic statistics.



\## Functional Requirements



\### 1. User Management

\- The user must be able to register.

\- The user must be able to log in.

\- The user must be able to update basic profile information.



\### 2. Reminders

\- The user must be able to create a reminder.

\- The user must be able to edit a reminder.

\- The user must be able to delete a reminder.

\- The user must be able to mark a reminder as completed.

\- The user must be able to assign a category.



\### 3. Reminder Categories

Initial categories:

\- Medical appointment

\- Medical exam

\- Bill payment

\- Vehicle inspection

\- Vehicle maintenance

\- Personal errand

\- Custom reminder



\### 4. Documents

\- The user must be able to attach a document to a reminder.

\- The user must be able to attach an image to a reminder.

\- The user must be able to view attached documents.

\- The user must be able to remove attached documents.



\### 5. Checklist

\- Each reminder must support a required items checklist.

\- The user must be able to add checklist items.

\- The user must be able to mark checklist items as completed.



\### 6. Notifications

\- The app must notify the user before the event.

\- The app should support configurable reminder times.



\### 7. Calendar

\- The user must be able to see reminders in a calendar view.



\### 8. Statistics

\- The app must show basic statistics:

&#x20; - Completed reminders

&#x20; - Pending reminders

&#x20; - Overdue reminders



\## Non-Functional Requirements



\### Security

\- User data must be protected.

\- Documents must be stored securely.

\- Users must only access their own data.



\### Performance

\- The app should load the home screen quickly.

\- Reminder creation should be simple and fast.



\### Availability

\- The app should be available for Android and iOS.



\### Scalability

\- The backend should support future growth.



\### Maintainability

\- The codebase should be organized and documented.



\## Out of Scope for MVP

\- Artificial Intelligence

\- Payment processing

\- Bank integrations

\- Family sharing

\- Advanced analytics

\- Outlook or Google Calendar integration

\- Web admin portal
