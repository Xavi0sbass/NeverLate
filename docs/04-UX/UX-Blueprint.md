\# NeverLate - Design Language



\## Product Vision



NeverLate is not a reminder application.



It is a Personal Life Management Platform that helps people stay organized, prepared, and confident before important life events.



\---



\# Experience Statement



Every interaction should reduce stress.



The application should never overwhelm the user.



Instead, it should answer one simple question:



"What do I need to do next?"



\---



\# Emotional Goals



Users should feel:



\- Organized

\- Prepared

\- Confident

\- Relaxed

\- In control



\---



\# User Interface Principles



\## One Primary Action



Each screen should have one obvious primary action.



Avoid multiple competing buttons.



\---



\## Minimal Cognitive Load



Show only the information needed for the current task.



Avoid clutter.



\---



\## Progressive Disclosure



Advanced options should remain hidden until needed.



\---



\## Clear Status



Every reminder should clearly indicate its current state.



Examples:



Upcoming



Today



Completed



Overdue



\---



\## Context First



Instead of showing raw data, show meaningful information.



Example:



Instead of:



Medical Appointment



Show:



Tomorrow at 9:00 AM



Dr. Rodríguez



You need:



• ID Card



• Insurance Card



• Previous Results
